# v0.6.0 — Milestone Audit: Réorganisation en accessors

**Audited:** 2026-06-19
**Status:** ✅ **passed**
**Score:** 11/11 requirements satisfied · 4/4 phases verified · 5/5 E2E flows complete · 0 critical gaps

into 5 new lazy accessors (`db.timescale`, `db.admin`, `db.maint`, `db.backup`, `db.schema`),
relocated 2 spatial-index methods to `db.spatial.*`, and kept every old flat name working as a
deprecated alias (removal scheduled v0.7.0). No new capability was added by design. **v0.6.0 is
already tagged and published to PyPI** (confirmed via the PyPI JSON API during Phase 24 verification).
