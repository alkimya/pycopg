<objective>
Build the pure, DB-free foundation of incremental ETL inside `pycopg/etl.py`, fully unit-tested
without a database. Three independent deliverables (ETL-INC-01):

1. `Pipeline.incremental_column` field + the module-level `_validate_incremental` helper wired into
   `__post_init__` (D-14..D-17).
2. The pure watermark-filter SQL builder `_build_incremental_extract_sql` (D-06..D-13).
3. The typed-JSONB-envelope serializers `_encode_watermark` / `_decode_watermark` (D-01..D-05).

Purpose: lay the unwired, isolated primitives that Phases 27 (run-log read/write) and 28 (extract
wiring, RunResult, async parity, docs) consume. This phase touches ONLY code paths that need no DB
connection — no `pipeline_runs` reads/writes, no `run()` wiring, no `Jsonb(...)` adapter wrap, no
async mirror, no docs. Those are explicitly Phases 27/28.

Output: 5 new symbols in `pycopg/etl.py` (one `Pipeline` field + four module-level helpers) plus
co-located DB-free tests in `tests/test_etl.py`.
