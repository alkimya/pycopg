**BUG-05: SRID inference explicit error (lines 1208-1213)**

Replace the silent SRID default in `from_geodataframe()`:

Current (lines 1208-1213):
```python
# Handle SRID
if srid is None and gdf.crs is not None:
    try:
        srid = gdf.crs.to_epsg()
    except Exception:
        srid = 4326  # Default to WGS84
```

Replace with explicit error handling:
```python
# Handle SRID — fail explicitly on unknown CRS instead of silently defaulting
if srid is None:
    if gdf.crs is None:
        raise ValueError(
            "GeoDataFrame has no CRS defined. "
            "Set gdf.crs or provide explicit srid parameter."
        )
    try:
        srid = gdf.crs.to_epsg()
        if srid is None:
            raise ValueError(
                f"Cannot determine EPSG code for CRS: {gdf.crs}. "
                f"Provide explicit srid parameter."
            )
    except ValueError:
        raise
    except Exception as e:
        raise ValueError(
            f"Failed to infer SRID from CRS {gdf.crs}. "
            f"Provide explicit srid parameter. Error: {e}"
        ) from e
```

This is a BREAKING CHANGE (allowed per v0.3.0 decisions): Previously, unknown CRS silently defaulted to SRID 4326. Now it raises ValueError, forcing users to either set a CRS on their GeoDataFrame or pass `srid=` explicitly.

Also note: The `except ValueError: raise` re-raise ensures our own ValueError from `srid is None` check propagates without being caught by the broader `except Exception`.
