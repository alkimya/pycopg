Overview paragraph explaining this is a consolidation release with one breaking change.

## Breaking Changes section:

### 1. GeoDataFrame CRS Validation (from_geodataframe)
- What changed: Raises ValueError on unknown CRS instead of silent SRID 4326 default
- Why: Silent defaults hide configuration errors, explicit errors prevent data corruption
- Who is affected: Anyone using from_geodataframe() with GeoDataFrames that have unknown or missing CRS
- Before (0.2.0): code example showing silent default behavior
- After (0.3.0): code example showing ValueError + fix with explicit set_crs() or srid parameter
- Impact: Low - only affects unknown CRS cases, EPSG-standard CRS like "EPSG:4326" still works automatically

## New Features section with code examples for:
- Full Async Parity (await db.to_dataframe, await db.vacuum, await db.pg_dump, await db.create_role, etc.)
- Retry/Backoff (automatic on Database.connect() and AsyncDatabase.connect())
- Statement Timeout (config.statement_timeout = 30000)
- Configurable Batch Size (db.insert_batch(table, rows, batch_size=500))

## Upgrade Checklist (markdown checklist):
- Review breaking changes
- Update from_geodataframe() callers to handle CRS explicitly if needed
- Optionally configure statement_timeout for query protection
- Optionally explore new async methods if using AsyncDatabase
- Run test suite to verify compatibility
