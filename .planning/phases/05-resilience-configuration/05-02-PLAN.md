  2. `test_connect_with_retry_retries_operational_error()` — Mock `psycopg.connect` to raise `psycopg.OperationalError` on first 2 calls, succeed on 3rd.   3. `test_connect_with_retry_does_not_retry_programming_error()` — Mock `psycopg.connect` to raise `psycopg.ProgrammingError`. Assert it raises ProgrammingError immediately without retrying. Assert psycopg.connect called exactly 1 time.

  4. `test_connect_with_retry_reraises_after_max_attempts()` — Mock `psycopg.connect` to always raise `psycopg.OperationalError`. Assert OperationalError is raised. Assert psycopg.connect called exactly 3 times (stop_after_attempt(3)).

  5. `test_insert_batch_uses_config_default_batch_size()` — This test needs a real DB. Using the existing test DB fixtures, create a Config with default_batch_size=2, create Database, create a test table, insert 5 rows via insert_batch (no explicit batch_size). Verify all 5 rows inserted (the default batch_size=2 means 3 batches internally, but total count should be 5). If no DB fixture exists, use `inspect.signature` to verify batch_size default is None.

  6. `test_insert_batch_explicit_batch_size_overrides_config()` — Use inspect.signature to verify batch_size parameter has default None, then verify calling with explicit batch_size=100 works.
