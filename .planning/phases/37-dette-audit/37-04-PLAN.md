|-----------|----------|-----------|-------------|-----------------|
| T-37-04-01 | Information disclosure | `TableNotFound` raise message in `truncate_table` | mitigate | The message names only the already-caller-supplied `f"{schema}.{name}"` — no DB internals, no other table names, no row data are leaked. `validate_identifiers(name, schema)` runs FIRST, so a malformed identifier raises the validation error (not TableNotFound) and never reaches the message-formatting path; the builder-pur invariant is preserved. |
| T-37-04-02 | Tampering | the new `table_exists` guard in `truncate_table` | accept | The guard is a read-only existence check that runs after identifier validation and changes no DDL semantics for existing tables; it only converts a raw psycopg `UndefinedTable` into a typed `TableNotFound` for missing tables. No new external input path. |
