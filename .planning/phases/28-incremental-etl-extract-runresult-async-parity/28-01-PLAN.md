<objective>
Wire the incremental watermark loop end-to-end in the **sync** `ETLAccessor`:
read the prior watermark before extract, apply `WHERE col > %s` via the
already-built `_build_incremental_extract_sql`, surface `watermark_used` and
`watermark_recorded` on `RunResult` (and thus `history()` / `last_run()`), and
make `dry_run` a faithful filtered preview. This is the foundation that the
async port (Plan 02) mirrors 1:1 and that the docs (Plan 03) describe.

Purpose: Close ETL-INC-03/-04/-07/-08/-09 on the sync side. The read half
(`_read_watermark`, Phase 27), the write half (`_end_run(watermark=)`, Phase 27),
and the filter builder (`_build_incremental_extract_sql`, Phase 26) all already
exist and are unit-tested — this plan connects the three.

Output: Updated `RunResult` dataclass (2 new fields), updated `_row_to_result`,
a shared filtered-extract path used by both the `run()` real path and the
`dry_run` fork, the sync incremental `dry_run` branch, and sync live-DB
integration tests.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | Injection (Tampering) | sync `run()` filtered extract — watermark value | mitigate | Watermark VALUE is always a bound parameter (`%s` from the builder, reconciled to a named `:wm` bind for `to_dataframe`); never f-string-interpolated. Asserted by the bound-param test (Task 3 #2) and SC-1. |
| T-28-02 | Injection (Tampering) | `_build_incremental_extract_sql` — column / source / schema identifiers | mitigate | Identifiers validated via `validate_identifiers` BEFORE any interpolation (builder already does this; the LIMIT-wrap must not introduce un-validated identifier interpolation). |
| T-28-03 | Information disclosure | `pipeline_runs.watermark` JSONB decode in `_row_to_result` | accept | Watermark is non-PII pipeline metadata; decode only reconstructs the typed scalar the pipeline itself wrote. Low value, no new surface. |

No new external surface (pure library; no network/auth/filesystem changes).
