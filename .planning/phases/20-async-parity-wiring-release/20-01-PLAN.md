<objective>
Mirror the locked sync `ETLAccessor` into a new `AsyncETLAccessor` class in the same `pycopg/etl.py` module, wire the lazy `async_db.etl` property on `AsyncDatabase`, and add the four ETL public symbols to the top-level package exports.

This is a straight async mirror. The sync surface is fully built and tested (Phases 17-19). Every pure module-level function (`_row_to_result`, `_is_sql_source`, `_build_insert_sql`, `_build_upsert_sql`, `build_truncate_sql`, `_step_label`, `build_init_sql`) is REUSED byte-for-byte — no duplication. The only structural differences from the sync accessor: every method becomes `async def`, every `with self._db.connect(autocommit=True)` becomes `async with`, every cursor `execute`/`fetchone`/`fetchall` gets `await`, the load block uses `async with self._db.session()` / `async with self._db.transaction()`, and the transform chain dispatches each step via `await asyncio.to_thread(step, df)` (SC-2).

Purpose: Close ETL-12 (async parity + asyncio.to_thread dispatch) and ETL-13 (lazy `async_db.etl` property + exports). After this plan, `async_db.etl.*` exists; two pre-existing `TestAsyncParity` failures go green automatically.
Output: `AsyncETLAccessor` class, `AsyncDatabase._etl` + `etl` property, four new top-level exports.
</objective>
