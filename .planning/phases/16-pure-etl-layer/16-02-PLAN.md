<objective>
Create the new `pycopg/etl.py` module ‚Äî the public flat `Pipeline` frozen dataclass
(D-01/D-02/D-03/D-09/D-11) plus the two pure SQL builders shipped this phase
(`build_init_sql()`, `build_truncate_sql()` ‚Äî D-10) ‚Äî and the DB-free unit-test

Purpose: This is the declarative object users write and the pure builder seam that
phases 17-19 build on. It mirrors `spatial.py` exactly: a module docstring stating
the security invariants, module-level stateless builders returning `(sql, params)`,
and `validate_identifiers(...)` called before any interpolation (D-13). The dataclass
is the public API contract for the whole milestone, so its field set and validation
are locked here (no `depends_on`/`schedule`/`retry` scope creep ‚Äî Pitfall 11).

Output: `pycopg/etl.py` (Pipeline + builders + an optional `_is_sql_source` helper),
`tests/test_etl.py` (DB-free tests).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pipeline definition / builder args ‚Üí generated SQL | Caller-supplied identifiers (`table`, `schema`) reach SQL text in `build_truncate_sql`. This is the single live injection surface in the pure layer. |

No I/O, no network, no auth, no session. SQL identifier injection is the dominant
(and essentially only) threat. Value injection is N/A: `build_truncate_sql` and
`build_init_sql` interpolate validated identifiers only and carry an empty params
list ‚Äî no value is ever interpolated this phase.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03 | Tampering | `build_truncate_sql(table, schema)` | mitigate | `validate_identifiers(table, schema)` is called BEFORE the `f"TRUNCATE TABLE {schema}.{table}"` interpolation (D-13); only validated identifiers are interpolated. Unit-tested: `build_truncate_sql('bad-name')` raises `InvalidIdentifier`. |
| T-16-04 | Tampering | `build_init_sql()` DDL | mitigate | Returns the static `ETL_INIT_PIPELINE_RUNS` constant verbatim ‚Äî no caller input interpolated, so no injection surface. `%s`-only constant (D-12). |
| T-16-05 | Spoofing/Tampering | `Pipeline.conflict_columns` | mitigate | Stored as an immutable `tuple` on a frozen dataclass; identifier validation of conflict columns is deferred to the load builders in Phase 18 (no identifier interpolation of conflict_columns happens this phase). Construction-time `ValueError` guards the upsert-without-key misconfiguration (D-07). |

Block-on severity: high. Zero new dependencies (stdlib `dataclasses`/`typing` +
existing pycopg only) ‚Äî no package install, so no `T-16-SC` supply-chain checkpoint
is required.
