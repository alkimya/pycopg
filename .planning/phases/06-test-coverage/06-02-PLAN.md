1. Create tests/test_migration_edge_cases.py (TEST-02):
   All tests use real PostgreSQL (db_config fixture) and temp_migrations_dir fixture.

   Create a helper fixture `migrator` that:
   - Creates a Database(db_config)
   - Creates a Migrator(db, temp_migrations_dir)
   - In teardown: drops the schema_migrations table if it exists, drops any test tables

   TestMigrationRollbackEdgeCases:
   - test_rollback_with_deleted_down_section:
     * Write migration file with UP and DOWN sections
     * Apply it (migrator.migrate())
     * Rewrite file removing the DOWN section (keep only UP)
     * Call migrator.rollback() — expect MigrationError with "No DOWN section" message
     * Cleanup: manually drop the test table

   - test_rollback_with_deleted_migration_file:
     * Write migration file, apply it
     * Delete the file from filesystem (path.unlink())
     * Call migrator.rollback() — expect MigrationError with "not found" message
     * Cleanup: manually drop the test table

   - test_rollback_with_syntax_error_in_down:
     * Write migration with valid UP but syntactically invalid DOWN ("DROP TABEL test_syntax;")
     * Apply it
     * Call migrator.rollback() — expect MigrationError (wraps SQL syntax error)
     * Cleanup: manually drop the test table

   - test_rollback_when_no_migrations_applied:
     * Don't apply any migrations
     * Call migrator.rollback() — should return empty list, not raise
     * Assert result == []

   - test_rollback_multiple_steps:
     * Write 3 migration files, apply all 3
     * Rollback with steps=2
     * Verify only 1 migration remains applied

   Important: Each test must clean up after itself. Use unique table names per test (e.g., test_rollback_del_down_{random_suffix} or just different names per test). Use try/finally in fixtures to ensure cleanup.

2. Create tests/test_session_edge_cases.py (TEST-03):
   All tests use real PostgreSQL (db_config fixture).

   TestSessionExceptionScenarios:
   - test_session_cleanup_after_exception:
     * Create Database(db_config)
     * Open session, raise ValueError inside
     * Catch ValueError outside
     * Assert db._session_conn is None
     * Assert db.in_session is False

   - test_nested_sessions_raise_error:
     * Open session
     * Inside session, try to open another session
     * Expect RuntimeError with "Already in session" message

   - test_session_survives_query_error:
     * Open session
     * Execute query on nonexistent table — catch the psycopg error
     * After catching, the session connection may be in error state
     * Note: psycopg3 requires rollback after error in transaction. Verify the session context handles this or document behavior.
     * The important test: after the error, the session context manager still cleans up properly (no leaked connection)

   - test_session_in_session_true_inside:
     * Verify in_session returns True while inside session block

   - test_session_conn_none_after_normal_exit:
     * Open session, do some work, exit normally
     * Assert _session_conn is None after exit

   - test_session_autocommit_mode:
     * Open session with autocommit=True
     * Execute DDL (CREATE TABLE), verify it persists without explicit commit
     * Cleanup: drop the table
