Create tests/test_database_integration.py with real PostgreSQL integration tests using the existing db_config fixture from conftest.py. These tests connect to pycopg_test database and exercise the core methods that make up the bulk of uncovered lines.

Test classes to create:

1. TestDatabaseCoreOperations:
   - test_connect_and_execute: Connect, execute SELECT 1, verify result
   - test_execute_with_params: Execute parameterized query
   - test_execute_autocommit: Execute with autocommit=True
   - test_cursor_context: Use cursor() context manager
   - test_insert_and_select: Create temp table, insert rows, select them back
   - test_insert_batch: Create temp table, insert_batch with list of dicts, verify count
   - test_upsert: Create table with unique constraint, upsert conflicting row
   - test_execute_returning_no_rows: Execute INSERT without RETURNING (returns empty list)

2. TestDatabaseSchema:
   - test_list_schemas: Verify 'public' in schemas
   - test_list_tables: Create temp table, verify it appears in list_tables
   - test_table_exists: Check table_exists returns True for known table, False for unknown
   - test_table_info: Create table, get table_info, verify columns returned
   - test_count: Create table with rows, verify count matches
   - test_schema_exists: Verify 'public' exists, 'nonexistent_xyz' does not

3. TestDatabaseSession:
   - test_session_basic: Open session, execute queries, verify results
   - test_session_in_session_property: Verify in_session is True inside session, False outside
   - test_session_connection_reuse: Verify queries in session use same connection (check pg_backend_pid)
   - test_session_autocommit: Session with autocommit=True works

4. TestDatabaseEngine:
   - test_engine_created_lazily: Engine is None until first access
   - test_engine_returns_sqlalchemy_engine: engine() returns Engine instance
   - test_to_dataframe: Create table with data, call to_dataframe, verify DataFrame shape (requires pandas)
   - test_from_dataframe: Create DataFrame, call from_dataframe, verify data in DB (requires pandas)

5. TestDatabaseDDL:
   - test_create_table: Create table with columns, verify it exists
   - test_drop_table: Create then drop table, verify gone
   - test_create_index: Create table, create index, verify in list_indexes
   - test_drop_index: Create then drop index

6. TestDatabaseAdmin:
   - test_size: Call size(), verify returns string (e.g. "8 MB")
   - test_table_sizes: Create table, call table_sizes, verify returns list of dicts
   - test_vacuum: Call vacuum on a table (just verify no error)
   - test_analyze: Call analyze on a table (just verify no error)

Important patterns:
- Use `db_config` fixture from conftest.py
- Create a `db` fixture that initializes Database(db_config) and connects
- Use a setup/teardown pattern: CREATE TABLE at start, DROP TABLE IF EXISTS at end
- Use unique table names per test (e.g., `test_insert_{uuid_suffix}`) or use a fixture that creates/drops
- Wrap table creation in try/finally to ensure cleanup
- For DataFrame tests, use pytest.importorskip("pandas") to handle optional dependency
