## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → schema/owner identifiers | new `schema`/`owner` params interpolated into CREATE EXTENSION / CREATE SCHEMA SQL |
| async engine lifecycle | engine holds pooled connections / credentials until disposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-09 | Tampering (SQL injection) | async create_extension `schema`, create_schema `owner` | mitigate | Mirror sync `if schema: validate_identifier(schema)` and `if owner: validate_identifier(owner)` before f-string interpolation. |
| T-11-10 | Information Disclosure / resource leak | async engine not disposed (was a no-op close) | mitigate | C2 fix disposes the engine and nulls the reference, releasing pooled connections (and their credentials) deterministically — mirrors sync close(). |
| T-11-11 | Tampering | C1 `add_primary_key` call | mitigate | `add_primary_key` (Plan 03) already validates `table`/`schema`/columns via `validate_identifiers`; C1 only forwards already-typed args. |
</threat_model>
