<objective>
Implement PAR-03: mirror the four async-only batch/streaming/notify methods into the sync `Database` class. Each is a direct mechanical mirror (D-01) of its async twin in `async_database.py`, swapping the async I/O pattern (`async with self.connect()` / `await`) for the established sync pattern (`with self.cursor(...)` / `fetchmany`, `self.execute`/`self.execute_many`).

Purpose: Restores sync↔async parity for `insert_many`, `upsert_many`, `stream`, `notify`. This plan touches only `database.py`, so it runs in Wave 1 in parallel with Plan 01 (config.py + async_database.py) — no file overlap.
Output: `Database.insert_many`, `Database.upsert_many`, `Database.stream`, `Database.notify`. `listen` is deliberately NOT mirrored (D-06).
</objective>
