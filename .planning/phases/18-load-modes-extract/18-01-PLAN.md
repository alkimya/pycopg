<objective>
Author the three pure, DB-free helpers in `pycopg/etl.py` that the Phase 18 `run()` body
(Plan 02) composes: `_build_insert_sql` (used by append + replace), `_build_upsert_sql`
(used by upsert; delegates to `_build_insert_sql` with an ON CONFLICT clause), and
`_step_label` (transform-step naming for `ETLTransformError`). These mirror the existing
`build_truncate_sql` pure-builder shape and borrow the SQL-construction + identifier-validation
logic of `base.py:_build_batch_insert_sql` and `database.py:upsert_many` â€” but as pure builders
that return `(sql, params)` for execution on a transactional connection, NOT methods that
acquire `self.cursor()` (RESEARCH Q1: the public methods are unusable inside `db.transaction()`).

Purpose: These builders are the only safe way to honor both SC-3 (atomic TRUNCATE+INSERT on one
connection) and SC-6 (validate_identifiers before interpolation) given the current `Database`
API â€” and being pure, they are fully unit-testable with no database, exactly like
`build_truncate_sql`.
Output: Two SQL builders + one label helper in `etl.py`, their DB-free unit tests in
`test_etl.py`, and ETL identifier-injection regression cases in `test_sql_injection.py`.
</objective>

<decisions_implemented>
- D-04: reuse the existing builders' SQL-construction + identifier-validation logic â€” as corrected
  by RESEARCH Q1 (option a), realized here as small PURE builders in `etl.py` rather than the public
  `insert_batch`/`upsert_many` methods (which crash inside a `db.transaction()` block). This is NOT
  the rejected speculative `build_upsert_sql` of D-04a; it is the minimal builder required because
  the public methods are unusable on the transactional connection.
- D-06: `_step_label` supplies the failing transform step's function name (with a `repr` fallback for
  lambdas/`functools.partial`) for the `ETLTransformError` message; the 1-based step-index message
  contract is locked by a unit test here.
- SC-6 (ETL-16 security clause): every new builder calls `validate_identifiers` on every identifier
  before any f-string interpolation; regression-tested against `EVIL_IDENTIFIERS`.
</decisions_implemented>
