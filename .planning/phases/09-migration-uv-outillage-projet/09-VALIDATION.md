# Phase 9 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> This is a tooling/CI/doc phase — most requirements are verified by **shell commands and CI runs**, not by new pytest tests. The existing pytest suite must continue to pass under `uv run pytest`.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest (already configured in `pyproject.toml` `[tool.pytest.ini_options]`) |
| **Config file** | `pyproject.toml` — `[tool.pytest.ini_options]` (`--cov-fail-under=70`) |
| **Quick run command** | `uv run pytest tests/ -x -q` |
| **Full suite command** | `uv run pytest` (picks up full `addopts` incl. `--cov-fail-under=70`) |
| **Estimated runtime** | ~30 seconds (unit); full suite needs a live Postgres |

---

## Sampling Rate

- **After every task commit:** Run `uv run pytest tests/ -x -q` (quick sanity; unit-level)
- **After every plan wave:** Run `uv run pytest` (full suite with coverage gate) — requires live Postgres
- **Before `/gsd-verify-work`:** Full suite green locally AND `tests.yml` CI green
- **Max feedback latency:** ~60 seconds locally; CI matrix ~5–10 min
