Create `pycopg/spatial.py` with a module docstring and `import json`, plus `from pycopg.utils import validate_identifiers`. Implement the internal resolver and one pure module-level builder per helper. Every builder takes explicit args (never `self`), returns `tuple[str, list]`, calls `validate_identifiers(table, schema, geom)` (and `validate_identifiers(*columns)` when `columns` given) as its first action, and emits ALL user values as `%s` placeholders appended to the params list. SRID is the only value interpolated directly, and only via `int(srid)` coercion (integer cast is injection-safe per RESEARCH Security Domain).

`_resolve_geometry(point=None, wkt=None, geojson=None, ref=None, srid=4326)` (D-05, D-07): require exactly one of point/wkt/geojson/ref (else `ValueError("Exactly one of point=, wkt=, geojson=, ref= must be provided")`). Returns the SQL fragment + params for the geometry input:
  - point=(x,y) → `f"ST_SetSRID(ST_MakePoint(%s, %s), {int(srid)})"`, params [x, y]
  - wkt="..." → `f"ST_GeomFromText(%s, {int(srid)})"`, params [wkt]
  - geojson={...} → `f"ST_SetSRID(ST_GeomFromGeoJSON(%s), {int(srid)})"`, params [json.dumps(geojson)]
  - ref=(ref_table, ref_col) → validate_identifiers(ref_table, ref_col), then return a sentinel the caller turns into an EXISTS subquery (D-08). The EXISTS form for a predicate helper is `EXISTS (SELECT 1 FROM {ref_table} AS _ref WHERE ST_{Predicate}(_ref.{ref_col}, t.{geom}))`. Exact sentinel shape is Claude's discretion — document it in the resolver docstring.

Builders to implement (SQL forms per RESEARCH "Helper SQL Patterns"; primary table aliased `t`):
  - `build_contains_sql(table, geom="geometry", schema="public", *, point=, wkt=, geojson=, ref=, srid=4326, columns=None, where=None, order_by=None, limit=None)` → `ST_Contains(t.{geom}, <geom_in>)` predicate; ref form uses EXISTS. Apply the `_build_select_sql` column/where/order_by/limit convention: `cols_str = ", ".join(columns) if columns else "*"`, spatial condition first, then `AND ({where})` if where, then ORDER BY / LIMIT.
  - `build_intersects_sql(...)` same signature shape → `ST_Intersects(t.{geom}, <geom_in>)`.
  - `build_within_sql(left_table, left_geom, right_table, right_geom, schema="public", columns=None, where=None, order_by=None, limit=None)` → two-table JOIN form `ST_Within(a.{left_geom}, b.{right_geom})` (Pitfall 6 / 08-DESIGN §3 — this helper keeps its dedicated join signature, NOT the geometry-input forms).
  - `build_dwithin_sql(table, geom, *, point=/wkt=/geojson=/ref=, srid, distance, unit="m", columns, where, order_by, limit)` → unit="m": `ST_DWithin(t.{geom}::geography, <geom_in>::geography, %s)`; unit="srid": `ST_DWithin(t.{geom}, <geom_in>, %s)`. distance appended as `%s`.
  - `build_distance_sql(...)` → scalar column `ST_Distance(...) AS distance`, unit="m" casts both sides `::geography`. Supports `order_by` (caller may order by the distance column).
  - `build_nearest_sql(table, geom, *, point=/wkt=/geojson=/ref=, srid, k=5, columns, where)` → KNN `ORDER BY t.{geom}::geography <-> <geom_in>::geography LIMIT %s` with k as `%s` (RESEARCH KNN decision: use `::geography` for metric ordering; note GiST index in docstring). No `unit=` param (D-10).
  - `build_area_sql(table, geom, *, unit="m", columns, where, order_by, limit)` → `ST_Area(t.{geom}::geography) AS area` (unit="m") or `ST_Area(t.{geom}) AS area` (unit="srid"). No geometry input (operates on the table's own column).
  - `build_perimeter_sql(...)` → `ST_Perimeter(...)`, same unit handling as area.
  - `build_centroid_sql(table, geom, *, columns, where, order_by, limit)` → `ST_X(ST_Centroid(t.{geom})) AS centroid_x, ST_Y(ST_Centroid(t.{geom})) AS centroid_y`. No `unit=` (D-10). Scalar result (no geometry column → into="gdf" forbidden, enforced in accessor in 14-02).
  - `build_buffer_sql(table, geom, *, distance, unit="m", columns, where, order_by, limit)` → `ST_Buffer(t.{geom}::geography, %s)::geometry AS buffer` (unit="m"); unit="srid" without the geography cast. Returns geometry → valid for into="gdf".
  - `build_transform_sql(table, geom, *, to_srid, columns, where, order_by, limit)` → `ST_Transform(t.{geom}, %s) AS geometry_transformed`, to_srid as `%s`. No `unit=` (D-10). Returns geometry → valid for into="gdf".

Every public-ish builder gets a numpydoc-shallow docstring (Summary + Parameters + Returns + Raises, NO Examples — Phase 13 convention; interrogate fail-under=95 counts these). Module docstring required. Cite the decision IDs in builder docstrings where relevant (e.g. dwithin "unit='m' casts ::geography per D-09").

Do NOT add the accessor classes here — they belong to Plan 14-02. Do NOT modify database.py / async_database.py / __init__.py here.
