## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → accessor method | Untrusted identifiers/values cross into builders, then into executed SQL against the live DB |
| accessor → database | Generated SQL + params executed via existing `execute` / `to_geodataframe` infrastructure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Tampering | accessor method inputs (table, geom, columns, geometry values) | mitigate | All SQL assembled by 14-01 builders which validate identifiers and parameterize values; accessors add no raw SQL |
| T-14-06 | Information Disclosure / Elevation | spatial query on a DB without PostGIS | mitigate | PostGIS guard: sync raises `ExtensionNotAvailable` at accessor construction; async raises on first method call (lazy `_postgis_ok`) |
| T-14-07 | Tampering | `into=` value | mitigate | Whitelist `rows`/`gdf`; scalar helpers reject `gdf` with `ValueError` before any SQL runs |
| T-14-08 | Tampering | `where=` raw fragment passed through to executed SQL | accept | Documented limitation matching `_build_select_sql`; `where_params=` deferred (D-11) |
| T-14-SC | Tampering | npm/pip/cargo installs | accept | Phase 14 installs zero packages (RESEARCH Package Legitimacy Audit: N/A) |
