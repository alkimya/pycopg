<objective>
Close B5 (SEC-04, decision D-04): three subprocess sites in `pycopg/database.py` (line 2169 in pg_dump, 2264 in pg_restore, 2283 in _psql_restore) build the child environment as `{**subprocess.os.environ, **env}`. `subprocess.os` is a fragile, undocumented re-export of the `os` module; the correct, explicit form (already used on the async side) is `os.environ`. Replace all three with `{**os.environ, **env}` and add a top-level `import os` to database.py (the module does not currently import `os`). Add a dedicated red->green regression test.

Purpose: Make the backup/restore subprocess env construction explicit and robust, matching the async reference implementation.
Output: Fixed `pycopg/database.py`, `tests/test_subprocess_env.py`.
</objective>
