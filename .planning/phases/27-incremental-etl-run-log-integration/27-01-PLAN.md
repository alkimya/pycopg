<objective>
Wire the Phase-26 typed watermark envelope through the **sync** run-log
persistence layer and prove four success-only persistence invariants against
a real PostgreSQL DB.

This phase adds exactly three things and proves them:
1. `_read_watermark(name)` — reads + decodes the last **successful, non-NULL**
   watermark for a pipeline (or `None`). Exists and is tested; **not yet
   applied as an extract filter** (that is Phase 28 / ETL-INC-03).
2. `_end_run(watermark=...)` success-only persist — binds
   `Jsonb(_encode_watermark(value))` only on the success path; failed/empty
   runs leave `watermark` NULL.
3. A **minimal** `max(col)` capture from the RAW extracted batch (before
   transforms) in sync `run()`, with per-type coercion and a D-06
   missing-column `ETLError` guard — so SC-1 is provable end-to-end.

Purpose: make the no-advance-on-failure, empty-batch-preserves, and
JSONB-round-trip invariants honestly provable now, while keeping the
WHERE-filter extract, `RunResult` fields, `dry_run`, async mirror, and docs
in Phase 28.

Output: 2 new SQL constants, 1 new method, 1 extended method, ~15 new lines
in `run()`, 1 new import, and 6 live-DB integration tests.
</objective>
