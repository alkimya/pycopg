**Group 1 — Maintenance/utility (add in a UTILITY section):**

1. `async def vacuum(self, table=None, schema="public", analyze=True, full=False) -> None` — Direct conversion of Database.vacuum (line 1613). Build VACUUM options (FULL, ANALYZE), table string. CRITICAL: Must pass autocommit=True to self.execute() because VACUUM cannot run inside a transaction.

2. `async def analyze(self, table=None, schema="public") -> None` — Direct conversion of Database.analyze (line 1633). Build table string, await self.execute() with autocommit=True.

3. `async def explain(self, sql, params=None, analyze=False, format="text") -> list[str]` — Direct conversion of Database.explain (line 1643). Build EXPLAIN options, execute, extract "QUERY PLAN" from result dicts. Note: this does NOT need autocommit.

**Group 2 — Backup/restore via subprocess (add in a BACKUP/RESTORE section):**

4. `async def pg_dump(self, output_file, format="custom", schema_only=False, data_only=False, tables=None, exclude_tables=None, schemas=None, compress=6, jobs=1) -> None` — Async conversion of Database.pg_dump (line 2034). Use `asyncio.create_subprocess_exec()` instead of `subprocess.run()`. Build command list identically to sync version. Pass PGPASSWORD via env dict: `env={**os.environ, **{"PGPASSWORD": self.config.password}}` if password exists. Use `await proc.communicate()` (NOT proc.wait()) to avoid deadlocks. Check proc.returncode, raise RuntimeError on failure. Import asyncio and os at top of method or file level.

5. `async def pg_restore(self, input_file, clean=False, if_exists=True, create=False, data_only=False, schema_only=False, tables=None, schemas=None, jobs=1, no_owner=False, no_privileges=False) -> None` — Async conversion of Database.pg_restore (line 2121). Same subprocess pattern as pg_dump. IMPORTANT: Handle .sql file detection — if input_file.suffix == ".sql" or not input_file.exists(), call `await self._psql_restore(input_file)` and return. Otherwise build pg_restore command and run async subprocess.

6. `async def _psql_restore(self, sql_file: Path) -> None` — Async conversion of Database._psql_restore (line 2216). Build psql command, run via asyncio.create_subprocess_exec(), same env/error handling pattern.

**Group 3 — CSV via async COPY (add in a CSV section or same BACKUP/RESTORE section):**

7. `async def copy_to_csv(self, table, output_file, schema="public", columns=None, delimiter=",", header=True, null_string="", encoding="UTF8") -> int` — Async conversion of Database.copy_to_csv (line 2235). Use `async with self.cursor() as cur:` then `async with cur.copy(...)` for COPY TO STDOUT. For file operations use `asyncio.to_thread()`: create parent dir, open file, write chunks, close file. Use try/finally for file cleanup. Return row count via SELECT COUNT(*).

8. `async def copy_from_csv(self, table, input_file, schema="public", columns=None, delimiter=",", header=True, null_string="", encoding="UTF8") -> int` — Async conversion of Database.copy_from_csv (line 2291). Use `async with self.cursor() as cur:` then `async with cur.copy(...)` for COPY FROM STDIN. Read file in chunks via `asyncio.to_thread()` with 8192 byte reads, write to copy. Return cur.rowcount.

**Important implementation notes:**
- All type hints must match Database signatures exactly. Use `str | Path` for file paths, `Literal["plain","custom","directory","tar"]` for format, `Optional[list[str]]` for optional lists.
- Import Path from pathlib at file level if not already imported.
- Import Literal from typing if not already imported (check — it may already be imported from Phase 2 work).
- For subprocess methods: use `asyncio.create_subprocess_exec(*cmd, ...)` NOT `create_subprocess_shell` (security — avoids shell injection).
- For CSV methods: use `asyncio.to_thread()` for all file I/O to avoid blocking the event loop.
- Docstrings follow Google-style with Args, Returns, Example sections matching Database.
